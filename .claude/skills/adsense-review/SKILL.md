# /adsense-review — 콘텐츠 발행 전 검수 SOP

## 원칙 (사용자 방침)
- **모든 자동 생성 글은 발행 전 반드시 검수한다.** "AI스럽지 않고, 맥락·법적으로 문제없고, 말이 되는지"를 본다.
- 자동: `engine/content/reviewer.py` 가 generate 단계에서 호출(통과분만 큐). 수동: 사람·Claude가 동일 루브릭으로 재확인.
- 의심되면 **보수적으로 미발행**(REJECT) → 재생성/수정.

## 검수 루브릭 (각 항목 PASS 해야 발행)

### 1. AI 티 / 자연스러움
- 클리셰·필러 금지: "In today's fast-paced world", "Whether you're A or B", "It's worth noting", "Look no further", "delve", "elevate", "robust", "seamless", "game-changer", 과한 em-dash·불릿 남발, 모든 문단이 같은 리듬.
- 사람 에디터가 쓴 듯 자연스럽고 구체적인가. 일반론·뻔한 말 채우기 X.

### 2. 사실 / 맥락 (말이 되는가)
- 주장이 그럴듯하고 내부 일관적인가(비교표 ↔ 본문 ↔ verdict 모순 없음).
- **휘발성 수치(정확 가격·벤치마크)를 단정하지 말 것** → "현재 가격은 공식 사이트 확인" 식 헤지. 날조된 구체 수치 금지(RESEARCH Caveat 3).
- 제품·기능 설명이 실제와 맞는가(명백한 오류 없음).

### 3. 법적 리스크 ⚠️
- **허위 1인칭 경험 주장 금지**: "직접 8주 써봤다 / After working with both / a joy to use" 등 **실제 안 한 테스트·체험을 단정하면 안 됨**(신뢰·표시광고 이슈). 자동 생성물은 "문서화된 기능·일반 워크플로 기준" 으로 표현.
- **명예훼손/허위사실 금지**: 경쟁 제품에 대한 검증 안 된 부정 단정 X. "최고/최저/1위" 등 절대 단정은 의견으로 표현하거나 근거 명시.
- **상표**: 제품명은 지시적 사용만(비교 목적 OK), 로고·사칭 X.
- **저작권**: 타 사이트 문장 복제 X(독창).
- **YMYL**(금융·건강·법률): 단정·조언성 주장 주의, 출처·면책.
- **제휴/광고 표시**: 제휴 링크 있으면 disclosure.

### 4. AdSense 정책
- 금지 주제·기만·클릭 유도 문구 없음. 독창적·실질 가치.

### 5. 논리 / 유용성 / E-E-A-T
- 검색 의도를 실제로 충족하는가. 섹션 흐름·결론이 일관.
- 출처가 주장을 뒷받침. 저자·작성/갱신일 존재.

## 출력 형식 (reviewer.py JSON / 사람 보고 공통)
```json
{ "passed": true|false,
  "severity": "none|low|medium|high",
  "ai_tells": ["..."],
  "issues": [{"type":"factual|legal|policy|coherence|ai_tone","detail":"...","fix":"..."}],
  "notes": "..." }
```
- **high(법적·허위·정책)** → 무조건 REJECT. medium → 수정 후 재검. low → 경고 첨부 발행 가능.

## 절대 금지
검수 없이 자동 발행, 허위 경험·허위사실 통과, 휘발성 수치 날조 통과, 명예훼손성 단정 통과.
