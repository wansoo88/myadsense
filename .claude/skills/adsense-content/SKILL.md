# /adsense-content — 고ROI 니치·키워드·콘텐츠 기획 SOP

## 언제 쓰나
- 니치 선정, 키워드/토픽 클러스터 리서치, 콘텐츠 캘린더 작성, 기존 글의 RPM 개선 기획.

## 원칙 (docs/RESEARCH.md F5~F7)
- **니치**: 금융·보험·기술·건강이 고CPC. **여행은 중간 단가(고CPC 아님)**(F6).
- **지역/언어**: Tier-1 영어권(미·영·캐) 검색 트래픽이 Tier-3 대비 ~5–30배 RPM(F5).
- **트래픽**: 오가닉 검색 ≫ 소셜 ≫ 유료(F7). 검색 의도가 곧 광고 단가.
- **승인·SEO 공통**: 글 수가 아니라 **품질·깊이·E-E-A-T**(F1). 절대 수치(R3~R9)는 신뢰 금지.

## 절차

### 1. 니치 확정 (가성비 = 단가 × 본인 역량)
- 고CPC 4대(금융·보험·기술·건강) 중 **본인이 1차 경험/지식 있는 하위 니치** 선택 → E-E-A-T 신호가 공짜로 확보됨.
- 하위 니치 예: 금융(세금·신용·핀테크), 보험(건강/자동차/여행자보험 비교), 기술(VPN·사이버보안·SaaS), 건강(질환별 정보·보조제 — YMYL 주의).
- ⚠️ YMYL(돈·건강) 주제는 E-E-A-T 요구가 높음 → 출처·전문성·최신성 필수.

### 2. 지역/언어 결정
- 가능하면 **영어권 검색 의도** 타깃(RPM 레버 가장 큼, F5).
- 한국어만 가능하면 한국 내 **고단가 하위주제**(세금·대출·보험비교·B2B SaaS)로 한정. (한국↔영어권 정확 배율은 불확실 → 자체 RPM-by-country로 검증, Open Q1)

### 3. 키워드·토픽 클러스터 리서치
- **검색 의도 유형 분류**: 정보형 / 비교형 / 거래형. **비교·거래형이 광고 단가↑**(구매 의도).
- 토픽 허브 1개 + 지원 글 N개로 **클러스터** 구성(내부 링크) → 권위·SEO 동시 강화.
- 키워드별 메모: 의도 / 추정 경쟁도 / 단가 신호(니치) / 본인 역량 매칭.
- ⚠️ 단가 절대값을 블로그에서 베끼지 말 것 → 니치·의도로 **상대 우선순위**만 판단.

### 4. 콘텐츠 계획 (품질 우선)
- 각 글: 1차 경험·데이터·도표 등 **독창적 가치** 1개 이상. 검색 의도를 끝까지 충족.
- 프로그래매틱 SEO(대량 자동생성)는 **신중** — helpful-content/E-E-A-T 집행 리스크(Open Q3). 페이지마다 실질 가치 없으면 금지.
- 발행 캘린더: "얇게 많이"가 아니라 **깊은 글 + 클러스터 확장**.

### 5. 측정·반복
- 발행 후 AdSense + Search Console에서 **글별 RPM·오가닉 세션** 기록.
- 고RPM 클러스터에 자원 집중(가성비). 저성과 주제는 통합/개선.

## 출력 형식
```
[adsense-content] 니치: {하위니치} / 타깃: {언어·지역}
■ 토픽 클러스터: 허브 "{...}" + 지원글 {N}개
■ 키워드(우선순위): 1){kw}[의도/단가신호] 2)... 
■ 콘텐츠 메모: 각 글의 독창 가치 1줄
■ 다음 액션 & 측정 지표(RPM·세션)
```

## 절대 금지
유료 트래픽 권유, 소셜 의존 권유, AI 대량 저품질 양산, 여행을 "고CPC"로 오안내, 블로그 CPC/RPM 절대값 단정.
