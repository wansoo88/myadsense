# /adsense-audit — 승인 준비도 · 정책 점검 SOP

## 언제 쓰나
- AdSense **신청 직전**, **거절 후 재신청 전**, 운영 중 **정기 정책 점검**.

## 가드레일 (먼저 — docs/RESEARCH.md F2·F3, 위반 시 계정 영구 정지)
- **무효 트래픽 금지**: 자기/지인 클릭, 봇, 클릭 교환, 유료 클릭 — 어떤 인위적 부풀리기도 금지. "테스트 클릭"도 금지.
- **Privacy Policy는 강제 필수**. 없으면 신청·운영 불가 판정.
- 절대 수치(RPM/CPC) 단정 금지 → 필요 시 사용자 AdSense 리포트로 검증.

## 절차

### 1. 필수 요건 점검 (Google 1차 기준, F1·F2)
- [ ] **18세 이상 + 사이트 HTML 편집 권한** 보유? (없으면 신청 불가)
- [ ] **Privacy Policy 페이지 존재 + 접근 가능?** 포함 항목 확인:
  - 데이터 수집·이용 공개 / 제3자(Google) 쿠키·광고 네트워크 고지 / 벤더 링크 / 맞춤광고 옵트아웃(Ads Settings · aboutads.info) / 개인정보보호법 준수.
- [ ] (권장·필수 아님, R2) About·Contact 페이지 — 신뢰 신호. 저비용이니 추가 권고하되 **거절 사유로 단정 금지.**

### 2. 콘텐츠 품질·권위 점검 (F1 — 글 수가 아니라 품질)
- [ ] 각 글이 **독창적**인가? (복제·스크래핑·AI 대량 양산·자동 번역 덤프 ❌)
- [ ] 검색 의도를 **실제로 충족**하는 깊이가 있는가? (얇은 글 다수보다 깊은 글)
- [ ] E-E-A-T 신호: 저자 정보·전문성·1차 경험·출처·최신성.
- [ ] 네비게이션·핵심 페이지 정상, 빈/공사중 페이지 없음.
- ⚠️ **"글 30~40개·1000단어" 같은 수치 기준은 강요하지 말 것**(R1, 공식 기준 없음). 다만 콘텐츠가 너무 빈약하면 "청중을 끌어들이는 충분한 양"이 안 된다고 안내.

### 3. 정책 위반 리스크 점검 (F3·F4)
- [ ] 금지 콘텐츠 없음(성인·폭력·저작권 침해·위험 행위 등).
- [ ] 클릭 유도 문구("광고를 클릭하세요") 없음.
- [ ] 무효 트래픽 이력/리스크 없음(과거 자기 클릭·유료 클릭·봇 유입).
- [ ] 유료/통제된 트래픽을 쓴다면 **referrerAdCreative(RAC) 준수**(2025-11-01, RSoC 한정, F4) — 기본은 유료 트래픽 회피 권고.

### 4. 거절 사유 매핑 (거절 후)
거절 메시지를 위 1~3 항목에 매핑 → 가장 흔한 실제 사유:
- "콘텐츠 가치 낮음/불충분" → §2 품질·깊이 보강.
- "정책 위반" → §3 항목별 제거.
- (Privacy Policy 누락) → §1 즉시 추가.
**14일 후 재신청** 전 audit 재실행.

## 출력 형식 (보고)
```
[adsense-audit] 결과: 신청 가능 / 보완 필요 / 신청 불가
■ 필수: Privacy Policy [O/X], HTML권한 [O/X], 18+ [O/X]
■ 품질: 독창성 [O/△/X], 깊이 [O/△/X], E-E-A-T [O/△/X]
■ 정책: 금지콘텐츠 [없음/발견], 무효트래픽 리스크 [낮음/높음]
■ 차단 사유(있으면): ...
■ 다음 액션: 1) ... 2) ...
```
필요 시 결과를 **로컬 HTML 보고서**로 저장(Artifact 금지 — CLAUDE.md 정책).

## 절대 금지
무효 트래픽 유발 제안, 반증된 통념(글 수·About/Contact 필수) 강요, 블로그 절대 수치 단정.
